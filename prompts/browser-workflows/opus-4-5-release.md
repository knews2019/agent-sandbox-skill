# Opus 4.5 Release

## Browser UI Testing

### User Story Workflow 1: Summarize the release

[] Understand the release
[] Take high level notes of the release
[] Take note of 5 key features of the release
[] Take note of 3 'hidden' features of the release
[] Report your understanding of the release to the user

### User Story Workflow 2: Screenshot every image on the page

[] Download every image on the page to a temp/browser-test-plans/opus-4-5-release/images/ directory
[] Read each image, understand the content, and then update the file name to represent the content of the image
[] Report the number of images found

### User Story Workflow 3: Get the pricing and performance metrics

[] Get the pricing and performance metrics - read text and images for the metrics
[] Report the pricing and performance metrics to the user

### User Story Workflow 4: Get the system card and summarize orchestration details

[] Find the system card link
[] Download the system card pdf into a temp/browser-test-plans/opus-4-5-release/system-card.pdf file
[] Use pdftotext (poppler) to extract text to system-card.txt
[] Search the txt file for 'orchestration', 'subagent', 'delegate', 'multi-agent' keywords
[] Search the txt file for 'long running', 'long duration', 'long', 'longer', keywords
[] Report 'long running' and 'orchestration' details as bullet points in two separate lists - focus on net new or enhanced capabilities

### User Story Workflow 5: Understand how to use each model 'haiku', 'sonnet', and 'opus'

[] Understand the release
[] Branch off to different webpages to understand how to use each Claude 4.5 model now that they're all available
[] Report your recommendations and understanding of each model to the user
